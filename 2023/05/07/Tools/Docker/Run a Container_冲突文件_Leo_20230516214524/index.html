Run a Container

Created2023-05-07|Updated2023-05-07|ToolsDocker
|Post View:

After pulling the cs61b/cs61b_pl_autograder:latest Docker image, you can run a container using the docker run command. Since this image is for an autograder, you’ll likely want to mount a volume to share files between your host machine and the container.

Here’s a basic example of how to use docker run to start a container with a mounted volume:

Replace /path/to/your/host/directory with the absolute path of the directory on your host machine that you want to share with the container. Replace /path/in/container with the desired path inside the container where you want the shared directory to be mounted. You can adjust the container name by changing cs61b_autograder to your preferred name.

Here’s a breakdown of the options used:

  • -it: Run the container in interactive mode with a TTY attached. This is useful if you want to interact with the container’s shell or the autograder command line interface.
  • --name: Assign a custom name to the container.
  • -v: Mount a volume from the host machine to the container. The format is host_directory:container_directory.

After running the container, you can use docker ps to list all running containers and confirm that your cs61b autograder container is up and running. Inside the container, you can then navigate to the mounted directory and run the autograder as needed.

after docker pull, what does the image locate

After you execute docker pull, the Docker image is downloaded and stored on your local machine. Docker images are stored in a location called the Docker image registry on your local machine. The specific location depends on the operating system you are using.

For Linux systems, Docker images are usually stored in /var/lib/docker. Within this directory, you’ll find various subdirectories like containers, volumes, image, network, etc. The actual images are stored in the image subdirectory, but they are not stored in a user-friendly format.

For macOS and Windows systems running Docker Desktop, images are stored within a virtual machine that Docker Desktop manages. Accessing the images directly on the filesystem is not as straightforward as on Linux.

However, you typically don’t need to access these image files directly. Docker manages the storage and retrieval of images for you. You can list all the images stored on your local machine using the following command:

This will display a table containing information about the images, including the repository name, tag, image ID, creation date, and size. To manage your images (e.g., remove or inspect them), you can use Docker commands such as docker rmi, docker image rm, or docker image inspect.
