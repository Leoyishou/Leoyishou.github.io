3. Collections

Overview

introduce me guava’s collection, and compare them to java’s collections

Guava is an open-source set of libraries for Java developed by Google. It contains several useful utilities and collections that extend the functionality of the Java Collections Framework. Some of these collection types are not found in the standard Java Collections Framework. Here are a few examples:

  1. Immutable Collections: Guava provides truly immutable versions of Collection, List, Set, Map, and SortedSet, among others, with simple, straightforward API. This is in contrast to Java’s Collections.unmodifiable* methods which create a view on the original collection and any changes in the original collection will reflect in the unmodifiable collection.

  2. Multiset: A Multiset is a type of Collection that supports order-independent equality and allows duplicate elements. This is different from a Set (which doesn’t allow duplicates) and a List (which has an order to its elements). Java doesn’t have a built-in direct equivalent of a Multiset.

  3. Multimap: A Multimap is similar to a Map, but it can map a single key to multiple values. Java’s Map interface maps a single key to a single value.

  4. BiMap: BiMap is a type of Map that allows bidirectional lookup between key and values. It maintains an inverse view, which is another BiMap containing reversed key-value pairs.

  5. Table: A Table is similar to a Map, but it can support two keys (like row key and column key) instead of one.

Comparing Guava collections with Java collections, Guava collections are feature-rich, more flexible and often more efficient. However, Java’s own collections have been enhanced over time, and with the introduction of functional programming in Java 8, many tasks that would have required Guava can now be done natively in Java.

Also, while Guava collections offer some unique features, they are not always necessary. The need to use them depends on the specific requirements of your application. For many general purposes, the collections provided by Java are enough. It is also worth noting that using Guava will add an external dependency to your project, which some teams might want to avoid.

Multiset

where is guava‘smultiset better than java’s list

The Guava Multiset interface is better suited than Java’s List interface for use cases where you need to count the number of occurrences of distinct elements.

Here are some reasons why:

  1. Ease of Counting: Multiset makes it easy to count the number of occurrences of each element. In contrast, you’d have to manually iterate over a List and keep track of counts yourself.

  3. Null Elements: Depending on your use case, the ability to include or exclude null elements might be an advantage. Most Multiset implementations allow null elements, while some List implementations do not (e.g., ArrayList does, but CopyOnWriteArrayList does not).

  4. No Duplicates: A Multiset does not duplicate its elements. If you add an element to a Multiset multiple times, the Multiset still only has one distinct copy of that element, and it keeps track of how many times you added it. A List would contain multiple copies of the element.

Multimap

Multimap’s substance

waht is the type of result of multimap.get(key)? is just it set?

The result of multimap.get(key) is a Collection<V>, where V is the type of the value in the Multimap.

The actual concrete type of the Collection returned by get() depends on the specific Multimap implementation. For example, for an ArrayListMultimap, it is an ArrayList, while for a HashMultimap, it is a HashSet.

In other words, the returned Collection can potentially allow duplicate values. This is one of the main features that distinguishes a Multimap from a regular Map. If you are using a SetMultimap, get(key) would return a Set<V>.

Do note that changes to the returned collection will update the underlying multimap, and vice versa.

In terms of usage, you can use the returned Collection just like any other Java Collection. For example, you can iterate over it, check its size, check whether it contains a certain value, etc.
