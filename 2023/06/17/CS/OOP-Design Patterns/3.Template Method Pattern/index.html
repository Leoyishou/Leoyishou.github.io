3.Template Method Pattern

Created2023-06-17|Updated2023-06-23|CSOOP-Design Patterns
|Post View:

Components

teach template method pattern for me === ANKI ===
![](Pasted%20image%2020230623120611.png)

The Template Method pattern is a behavioral design pattern that defines the skeleton of an algorithm in a superclass but lets subclasses override specific steps of the algorithm without changing its structure. The pattern allows subclasses to redefine certain steps of an algorithm without changing the algorithm’s structure.

The key elements of this pattern are:

  1. Abstract Class: This class defines the template method that outlines the algorithm. The template method consists of calls to primitive and hook operations. It may be declared as final to prevent subclasses from modifying the sequence of steps in the algorithm.

  2. Primitive Operations: These are abstract methods that must be implemented by each concrete subclass to fulfill specific algorithm steps.

  3. Hook Operations: These are methods that provide default behavior that can be optionally overridden by subclasses.

Let’s consider an example to understand it better. Let’s say we have a SocialMedia abstract class which contains the method post, a common method for posting on any social media platform:

In the SocialMedia abstract class, post is the template method. This method includes the common steps to post content on any social media platform. It calls the authenticate, createPost, and publish methods. The authenticate and createPost methods are abstract and must be implemented by subclasses. The publish method is a hook operation with a default behavior.

Now let’s create two subclasses, Facebook and Twitter, which will implement authenticate and createPost:

In this way, the subclasses Facebook and Twitter can provide their own implementation of the authenticate and createPost methods while adhering to the template provided by the SocialMedia abstract class.

You would use the Template Method pattern when you want to:

  • Implement the invariant parts of an algorithm once and let subclasses implement the behavior that can vary.
  • Localize the common behavior among subclasses and let subclasses just extend that behavior.
  • Control subclasses extensions. You can define a template method that calls “hook” methods at specific points, thereby permitting extensions only at those points.
