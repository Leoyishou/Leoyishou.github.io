what the place do lambda replace? is it a instance?

Lambda expressions in Java are used primarily to replace instances of anonymous inner classes that have a single method. They allow you to write more concise and readable code by eliminating boilerplate.

So, lambda expressions can be thought of as a kind of “anonymous function” – they don’t have a name, but they have a parameter list, a body, a return type, and also possibly a list of exceptions that can be thrown. But unlike methods, lambdas can be passed around as if they were objects (hence the term “first-class functions”).

System.out::println what’s this? is it general syntax in java like lambda, or is it special syntax just for stream method?

The syntax System.out::println is an example of a method reference in Java, introduced in Java 8 along with lambda expressions. It is a shorthand notation for a lambda expression that calls a specific method.

In this case, System.out::println is equivalent to the lambda expression (x) -> System.out.println(x). Both represent a function that takes an input (in this case, any object), and calls System.out.println on it.

Method references can be used anywhere a functional interface is expected, which is often but not exclusively in the context of operations on streams. They are part of the general syntax of Java, not just for stream methods.

Here’s how it works:

  • System.out is an instance of PrintStream.
  • println is a method of PrintStream.

The :: operator is used to reference the println method of the PrintStream instance System.out.

There are four kinds of method references:

  1. Reference to a static method: ContainingClass::staticMethodName
  2. Reference to an instance method of a particular object: instance::instanceMethodName
  3. Reference to an instance method of an arbitrary object of a particular type: ContainingType::methodName
  4. Reference to a constructor: ClassName::new

The System.out::println method reference is of the second type. It’s referring to an instance method (println) of a particular object (System.out).

why? Variable used in lambda expression should be final or effectively final

In Java 8, lambdas can only access local variables that are final or “effectively final”. An effectively final variable is one that is not modified after it is initialized. In other words, it’s a variable that could be declared final but it doesn’t have to be.

The restriction exists because of how lambdas are implemented in Java. When you create a lambda, it’s an instance of a functional interface, and when you access a variable from the enclosing scope, it’s actually passed to the lambda when it’s instantiated. If you were allowed to modify that variable, it would create problems because the lambda might be used in a different thread, and changes in one thread might not be visible in another. This is a common source of bugs in multi-threaded programming.

To ensure that the code behaves as expected and to prevent multi-threading issues, Java simply disallows modification of local variables accessed from within lambda expressions.

where can I use :: except for stream?

The method reference syntax :: can be used anywhere you are expected to provide a functional interface instance. This is not limited to streams. Here are some examples:

1. Thread Start:

You can use method references instead of lambda expressions when starting a thread.

2. Event Handling:

In JavaFX or Swing, you can use method references in event handling.

3. Optional.ifPresent():

Optional.ifPresent() takes a Consumer as an argument, which can be expressed with a method reference.

4. Custom Functional Interfaces:

If you define your own functional interfaces, you can also use method references.

5. Comparators:

You can use method references when defining comparators for sorting.

In all of these examples, the double colon operator (::) is used to refer to a method without invoking it. This is the main usage of method references. They can be used in any context where you need to supply a function that fits the signature of a method that already exists.
