regex syntax

introduce java’s regex

Java’s Regular Expressions, or “regex”, is a powerful tool for manipulating strings. Regex is essentially a tiny, highly specialized programming language embedded inside Java and many other languages. You can use it in code to specify search patterns in text.

Regular expressions use a combination of literal characters and special symbols to describe patterns. Here is a brief overview of some common regex syntax:

  • .: Matches any single character except newline.
  • *: Matches zero or more of the preceding character/group.
  • +: Matches one or more of the preceding character/group.
  • ?: Matches zero or one of the preceding character/group.
  • \\d: Matches any digit, equivalent to [0-9].
  • \\D: Matches any non-digit character.
  • \\s: Matches any whitespace character (space, tab, newline, etc).
  • \\S: Matches any non-whitespace character.
  • \\w: Matches any word character (a-z, A-Z, 0-9, _).
  • \\W: Matches any non-word character.
  • []: Denotes a character class, matching any single character within the brackets.
  • ^: Inverts the character class when used as the first character within [], matches the start of a line when used outside [].
  • $: Matches the end of a line.
  • {n}: Matches exactly ‘n’ occurrences of the preceding character/group.
  • {n,}: Matches ‘n’ or more occurrences of the preceding character

regex API

the common usage of regex package

Java’s java.util.regex package provides the following classes:

  1. Pattern: A compiled representation of a regular expression.

  2. Matcher: An engine that performs match operations on a character sequence by interpreting a Pattern.

  3. PatternSyntaxException: Unchecked exception thrown to indicate a syntax error in a regular expression pattern.

demo

explain !COMMENT_PATTERN.matcher(line).find(); for me

The line of code !COMMENT_PATTERN.matcher(line).find(); is a part of the logic for checking if a line in a text file is a comment line or not.

Let’s break it down:

  • COMMENT_PATTERN is a Pattern object in Java, which is a compiled version of a regular expression. It is used to define the pattern for a comment line in the code.

  • COMMENT_PATTERN.matcher(line) creates a Matcher object that can match the given line against the pattern. This Matcher can be used to perform various match operations against the regular expression.

  • .find() is a method from the Matcher class, it attempts to find the next subsequence of the input sequence that matches the pattern. In other words, it scans the line from the beginning to see if there’s a match to the COMMENT_PATTERN.

  • The ! operator is a logical compliment operator in Java, it will invert the value of a boolean.

So overall, !COMMENT_PATTERN.matcher(line).find() will return true if the line does not match the COMMENT_PATTERN, and false if it does. This is used to indicate whether a line of code is a comment or not according to the defined pattern. In the context of the code, it’s used to filter out comment lines.
